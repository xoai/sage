## THE NEXT THING TO DO: the L-series

**Sage's central claim has no evidence behind it.**

All 11 eval scenarios are single-session. `/continue` (resume fidelity),
cross-session memory recall, the decision log outliving a context window,
`/reflect`'s cross-cycle value — **zero scenarios**. That is the differentiated
60% of the product, and `coverage.yaml` carries the rows saying so:

    workflow-continue          "Nothing measures resume fidelity. This is the
                               headline long-horizon claim, and it is unmeasured."
    capability-session-bridge  "Writes the manifest /continue reads — the literal
                               mechanism of the long-horizon claim."

Everything shipped in v1.3.0 sharpened the mechanical spine, and the evals confirm
it works. But the reason anyone would *choose* Sage over a bare frontier agent is
the long-horizon story, and it is the one claim we have no right to make.

**Spec:** `.sage/docs/sage-upgrade-harness/14-spec-phase5-evidence2.md` (ADR-13,
R120–R122). L1 = resume fidelity. L2 = memory recall shapes a later session.
L3 = scope-hold across a long plan (stretch).

The harness needs a multi-session driver: N sessions against **one persistent
fixture**, same deterministic graders, both conditions. `develop/evals/run_evals.py`
currently builds a fresh workspace per run — that is the thing to extend.

## The codex / opencode ports — one probe from finished

`docs/platform-veto-investigation-2026-07-12.md` has the full account. Short
version:

**Both platforms CAN enforce.** Neither is proven, so both are honestly `false`
and Tier C.

- **Codex**: its shipped binary contains the whole hook system — and this string:
  `PreToolUse hook exited with code 2 but did not write a blocking reason to stderr`.
  That is **Sage's contract exactly**. Sage's *unmodified* `sage-spec-gate.sh` was
  verified to exit 2 on a codex-shaped payload. **Blocked on:** finding the config
  that activates the feature (7 attempts fired zero hooks). **Best lead:** the
  binary references `CLAUDE_PLUGIN_ROOT` and `hooks/hooks.json` — codex appears to
  consume Claude-Code-style plugins, and **Sage already ships one in that exact
  layout.** That may be the entire port.

- **OpenCode**: plugin loads ✓, `tool.execute.before` fires ✓, Sage's gate runs
  inside it ✓, and — verified offline — the patch parser extracts the path from an
  `apply_patch` body ✓ and the gate exits 2 on it ✓. **The only unobserved link is
  whether opencode honours the `throw` by blocking.** One clean probe.

## Traps set for you

**1. The attestations expire at release 1.4.** Two claude-code capabilities
(`native-skill-discovery`, `pre-tool-veto` inside subagents) are `attested`, not
`true`. **The moment 1.4 is cut, conformance goes RED** until someone re-probes
them. That is the mechanism working — it stops a platform claim outliving its
evidence — but it will look like a spurious failure. The probes are already
written (`develop/conformance/probes/`); it is a re-run, not a re-investigation.

**2. E11 is 2/3, not 3/3.** Description-triggered skill delivery is
**probabilistic**. That is the trade ADR-9 made. Do not "fix" the docs by rounding
it up.

**3. The eval harness has never installed the plugin.** It drives `bin/sage init`.
So the path most users actually take is structurally untested — that is precisely
how `sage-navigator` shipped a routing table a release out of date for two
releases. The overlay is a mapped surface now, but every row is `uncovered:`.

## The lesson this release cost the most to learn

Four separate times, a mechanism caught what prose and good intentions missed:

- The **eval suite** proved the diet cost no behaviour — then found that the
  **ledger**, the entire evidence base for "every task was independently
  reviewed", was being produced by the model's *goodwill*. In two runs of three it
  simply was not written, and those runs looked exactly like cycles that had done
  their work. `ledger.py` generates it now. E9 went 1/3 → 3/3.
- The **conformance suite** caught a regression this same programme had shipped
  two phases earlier (the diet stranded 220 lines on four community platforms).
- The **coverage contract**, once pointed at the plugin overlay, exposed the blind
  spot that let the navigator rot in public.
- **Instrumentation** stopped me writing `pre-tool-veto: true` for codex on the
  strength of documentation that matches Sage's contract almost perfectly — but
  which no real session ever honoured.

Twice, a naive check would have reported success when *nothing had happened at
all*: codex's read-only sandbox blocked an edit (I would have recorded a veto);
opencode's crashed runs left the file unchanged (same). **Nothing-happened and
correctly-blocked are indistinguishable unless you instrument the mechanism
itself.**

> **Test the instrument before you trust the measurement.**
> A grader is code. It has bugs like code. Only reality finds them.
